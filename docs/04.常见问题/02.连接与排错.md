这篇连接与排错说明当你准备连接 MTNetwork 但遇到常见问题时，应该怎样按层排查。

## 先判断是哪一层问题
把问题先分成四类，会比一上来就问“为什么进不去”更有效：

- 文档层：你还没搞清信息入口和接入流程。
- 主站层：实时连接信息、页面状态或入口本身有变化。
- 客户端层：版本、模组、资源包、账户绑定或输入方式有问题。
- 网络层：运营商、IPv4 / IPv6、代理或本地网络导致连接失败。

## 连接失败时的检查顺序
1. 先打开主站，确认当前连接页与实时状态是否正常。
2. 再确认自己使用的是 `Java` 还是 `Bedrock`，不要混用入口。
3. 关闭非必要模组、代理和整合包，尽量回到最小化环境测试。
4. 如果主站明确显示开放，但你仍然无法连接，再检查网络与客户端信息。

## Java 与 Bedrock 的常见差异
| 场景 | 优先检查什么 |
| :--- | :--- |
| Java 无法进入 | 版本号、客户端本体、模组冲突、网络出口 |
| Bedrock 无法进入 | 当前接入参数、平台差异、网络限制、是否需要额外绑定 |
| 两端都异常 | 主站状态、连接页、服务端维护公告、网络环境 |

## 需要反馈时带上什么信息
正式反馈时，至少准备这些内容：

- 客户端类型（Java / Bedrock）
- 具体版本
- 是否使用模组、整合包或代理
- 看到的报错文本或截图
- 你访问的主站页面状态
- 问题发生的大致时间

如果你提供的信息不足，别人很难帮你把问题和“主站变化”“客户端问题”“网络问题”区分开。
