## 先自行确认问题范围
在正式反馈之前，先做最基本的自查：

- 这是不是你自己的客户端、网络、模组或输入方式导致的问题。
- 这个现象是否已经被别人反馈过。
- 你能否稳定复现，还是只出现过一次。
- 它影响的是连接、规则、安全，还是某个具体功能。

如果你还不确定怎么描述问题，建议先读一遍 [提问的艺术](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way/blob/main/README-zh_CN.md)。

## BUG 反馈怎么写
当你确认问题存在时，尽量提供：

- 出现时间
- 客户端类型（Java / Bedrock）
- 相关版本、模组或资源包
- 复现步骤
- 截图、日志或录屏
- 你原本期望发生什么，实际又发生了什么

如果问题需要进一步跟踪，请提交到服务器的 [GitHub Issues](https://github.com/txyyddss/txmcsv/issues)，这样更容易被持续记录与回看。

## 功能建议怎么提
- 可以直接在 [GitHub Issues](https://github.com/txyyddss/txmcsv/issues) 提交建议。
- 不要求你把模板每一格都写满，但至少要把目标、场景和预期影响说明白。
- “天马行空”不是问题，模糊和无法理解才是问题。
- 建议最终是否采纳，不代表其价值高低；重点是它是否适合当前服务器方向。

## 哪些情况不适合直接开 Issue
- 只是连接参数找不到，先看主站和 [连接与排错](/faq/troubleshooting)。
- 只是规则边界不确定，先回看 [基础规则](/getting-started/rules) 和 [社区与安全](/player-guide/community-safety)。
- 只是一次性的小疑问，先在QQ群组里简短确认。
