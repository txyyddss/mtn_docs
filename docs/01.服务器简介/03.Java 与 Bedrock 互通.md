这篇互通说明介绍 Java 与 Bedrock 同服具体意味着什么、哪些体验是一致的，以及哪些差异仍然存在。

## 互通具体意味着什么
- `Java Edition` 与 `Bedrock Edition` 进入的是同一个长期生存世界。
- 世界中的公共设施、建筑历史、玩家活动与主站公开记录围绕同一台服务器展开。
- 文档站会统一使用 “Java / Bedrock 同服” 这个表述，而不是把它们当成两套彼此独立的服务器。

## 哪些体验应当被理解为一致
- 两端玩家面对同一张地图、同一套公共空间与同一份长期历史。
- 主站上的部分公开记录会把两端玩家放在同一叙事框架下理解。
- 规则、公共设施礼仪、社区边界和反馈渠道对两端玩家是共通的。

## 哪些差异仍然存在
- 控制方式、界面表现和部分交互手感不会完全相同。
- 某些客户端能力、资源包或菜单体验可能会有平台差异。
- 当你看到文档说“基本一致”时，应理解为公共体验一致，而不是每一个按钮和视觉都完全相同。

## 账户绑定与数据理解
- 服务器支持 Java 与 Bedrock 账户绑定，用于在需要时把两端进度与身份理解到同一玩家身上。
- 绑定流程与相关命令，请以服内帮助和 [服务器指令](/player-guide/commands) 中的说明为准。
- 当主站或文档提到“共享进度”时，前提通常是该玩家已经按照当前策略完成绑定。
