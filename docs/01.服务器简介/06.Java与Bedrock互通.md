## 互通概览

MTNetwork 支持 **Java Edition（JE）** 与 **Bedrock Edition（BE）** 双版本互通。无论你使用哪个版本，都可以在同一个世界中和其他玩家一起游玩。

互通由服务端转译层实现，BE 玩家的客户端数据包会被实时转换为 JE 格式。大多数核心玩法（建造、生存、交互）都能正常工作。

## 账号绑定

JE 与 BE 账号之间可以进行绑定，绑定后数据将共享。

| 指令 | 说明 |
| :--- | :--- |
| `/linkplayer <对方账户名>` | 向目标账号发起绑定请求 |
| `/linkplayer <对方账户名> <验证码>` | 输入验证码确认绑定 |
| `/unlinkplayer` | 解除当前绑定 |

::: tip 绑定方向
绑定请求需要双方分别操作。发起方使用第一条命令，接收方会收到验证码，然后用第二条命令确认。
:::

## BE 玩家注意事项

- **操作入口**：BE 客户端没有聊天命令补全，推荐使用 `/bgui open main` 打开图形化菜单。
- **皮肤**：BE 端的自定义皮肤在 JE 玩家视角中可能显示异常，可使用 `/skin` 系列指令设置通用皮肤。
- **资源包**：服务器可能会下发自定义资源包，首次连接时请允许下载。
- **红石差异**：JE 与 BE 的红石机制存在差异，部分复杂红石装置可能仅在 JE 端正常工作。

## 已知限制

- 部分 JE 特有的交互方式（如副手、战斗冷却）在 BE 端的表现可能不完全一致。
- 地图画和自定义合成在跨版本场景下可能存在显示差异。
- 如遇到互通相关的异常行为，按 [反馈问题](/player-guide/report-issue) 流程提交。
