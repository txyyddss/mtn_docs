这篇服务器简介介绍 MTNetwork 的定位、公开叙事以及文档与主站各自负责什么。

## 这是一台怎样的服务器
- 以长期生存为核心，不做赛季重置和快餐式起号。
- 原版优先，只接受少量服务于公平、互通和公开记录的功能增强。
- 鼓励公共工程、长期建筑和多人协作，而不是把世界理解成一次性消耗品。
- Java 与 Bedrock 进入同一世界，玩家历史与公开记录围绕同一服务器展开。
- 非营利运营，不提供付费特权，也不把文档写成营销页。

## 这不是哪一类服务器
- 不是快餐服。这里强调长期参与、世界历史和后来者可持续使用的公共空间。
- 不是商业服。主站、文档和公开记录的存在，是为了透明和可理解，不是为了售卖捷径。
- 不是强制度量服。排行榜与记录页用于公开说明，不等于用单一指标定义玩家价值。
- 不是纯后台说明书。文档主要面向玩家与访客，而不是只给管理组使用。

## 为什么文档和主站要分开看
文档站负责解释“这是什么”“为什么这样设计”“应该怎样理解公开页面”，主站则负责展示“现在公开可见的状态是什么”。

- 文档站更适合读定位、规则、边界和阅读说明。
- 主站更适合看实时状态、玩家档案、排行榜、热力图和画廊入口。
- 任何会变动的信息，例如连接参数、实时在线状态和当前公开页面结构，都应以主站为准。

## 发展历程概览
| 阶段 | 时间 | 重点变化 |
| :--- | :--- | :--- |
| 起步阶段 | 2024 | 从好友间的封闭基岩版世界起步，随后迁移到 Java 侧并逐步开放。 |
| 深度迭代 | 2024 - 2025 | 经过多轮服务端、玩法和性能调整，逐步形成“长期生存 + 公开记录”的方向。 |
| 现代化建设 | 2025 | 引入 GitHub Issues、自动化发布流程与更完整的公开协作方式。 |
| 重构阶段 | 2026 至今 | 回归原版优先与极致性能，主站和 VitePress 文档体系逐步成形。 |

现有长历史、版本迁移和技术路线，应该理解为“这台服务器为什么会变成今天这个样子”，而不是对每一个旧机制的永久承诺。
