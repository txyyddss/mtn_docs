这篇加入方式说明新玩家在连接 MTNetwork 前需要确认的信息来源、接入顺序与变化处理方式。

## 先区分文档与主站
请先把“文档站”和“主站”分开理解：

- 文档站负责解释流程、规则和页面含义。
- 主站负责展示实时状态、当前连接信息和公开记录入口。
- 任何会变化的内容，例如连接参数、开放策略、在线状态或入口页面，都以主站为准。

## 加入前需要确认的事
### 客户端类型
- `Java Edition` 与 `Bedrock Edition` 都围绕同一世界活动。
- 两端的输入方式和部分 UI 不同，但世界历史、公共建筑和公开记录是共享的。

### 网络与连接
- 实时连接参数请直接在主站查看，不在文档中写死。
- 由于服务器采用家庭网络与打洞相关的接入方式，`IPv4` 端口可能发生变化。
- 如果主站提供 `IPv6`、备用入口或其他接入说明，应优先遵循当时页面。

### 账户与策略
- 文档不预告尚未公开的审核、白名单或认证细节。
- 如果主站连接页要求先登记、绑定或完成额外步骤，以当时页面提示为准。

## 建议接入流程
1. 先读 [服务器简介](/getting-started/server-intro)，确认自己加入的是长期生存社区。
2. 再看主站，确认当前连接页、实时状态和开放情况。
3. 根据自己的客户端类型，选择 `Java` 或 `Bedrock` 侧的接入方式。
4. 首次进入前补读 [基础规则](/getting-started/rules) 与 [社区与安全](/player-guide/community-safety)。
5. 如果你已经准备连接但遇到问题，直接跳到 [连接与排错](/faq/troubleshooting)。

## 信息发生变化时怎么办
- 文档优先说明原则，不承诺实时参数永远不变。
- 如果文档与主站出现不一致，优先相信主站上的实时连接信息。
- 如果主站本身显示异常或入口失效，再按照 [反馈问题](/player-guide/report-issue) 中的流程反馈。
